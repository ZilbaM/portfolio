Schedule

  • PHP
  • Backend
  • Data Parsing

Overview

To improve in PHP, I decided to make my own schedule, by extracting my timetable from the intranet of my school, then sort the useful data and parse it on my page. The hard part was documenting on the data sorting, and adapting the different existing methods to my project. The design page wasn't something I cared about, so I made a simple neumorphic design for the different event of my schedule. Using my experience, i am currently working on integrating this concept to a chrome plugin, to access my schedule anytime and send notifications.

Technical process

The extracting of the data was done through a RSS data feed already in place on the intranet of my school. I used the link laying in the intranet's code source to load the data, which was in the .ics format. Then, I used Regular Expressions (Regex) to cut through the useless data and extract the element i needed in an array. The parsing was basic loops and HTML.
